# ☕️ EzoCafe Log (エゾカフェ・ログ)

## 📖 プロジェクト概要
既存SNSの「いいね」や「フォロワー数」による精神的疲弊から脱却し、純粋なカフェ巡りの記録と、ユーザー自身の「足跡の蓄積」に特化したプロダクトです。
コンセプトは **「静寂」「自衛のDX」「記憶の地層」**。
ユーザーには承認欲求を煽らない心理的安全性の高い秘密基地と、圧倒的なサクサク感を提供しつつ、裏側ではAIと匿名UUIDを駆使して、将来のB2Bビジネス（カフェ経営者向けデータ基盤）に繋がる高純度のデータベースを自動精製します。

---

## 🛠 技術スタック (完全サーバーレス・エッジアーキテクチャ)
- **フロントエンド:** HTML / CSS / Vanilla JavaScript / Canvas API (画像圧縮＆マップ画像化)
- **バックエンド:** Cloudflare Workers
- **データベース:** Cloudflare D1 (SQLite)
- **AIモデル (Cloudflare Workers AI):**
  - `llama-3.2-11b-vision-instruct` (画像解析・不適切画像の自動破棄)
  - `llama-3.1-8b-instruct` (テキスト解析・多次元タグのJSON構造化・状態変化の検知)
- **外部連携API / ツール:**
  - `html2canvas` (マップのウォーターマーク付き画像出力)
  - `exifr` (Exif情報抽出)
  - `Open-Meteo API` (過去・現在の気象データ自動取得)
  - `Leaflet.js` + `OpenStreetMap` (マップ描画・LOD UI制御)
  - `Chart.js` (データ分析・可視化)
- **バージョン管理:** Git / GitHub (`.gitignore` による環境分離済み)

## ✨ コア機能と極上DX体験（フェーズ1.8 完成版）

### 1. 段階的開示UI (Progressive Disclosure) と神速入力
- **画像ファースト入力:** 画面を開いた瞬間は「写真を追加」のみ。写真を選ぶとフワッと入力フォームが展開し、ユーザーの認知負荷を極限まで下げます。
- **タイムトラベル天気＆位置取得:** 写真のExifデータから「撮影日時」「位置情報」を抽出し、気象APIから**当時の天気と気温を自動入力**。
- **一括「未整理」ストック:** 複数枚の写真を同時に選ぶと、裏側で「📦未整理」の空日記として一瞬で一括ストックされる非同期処理を実装。

### 2. LOD（遅延レンダリング）搭載の超軽量マップ
- **北海道完全ロック:** マップの表示範囲を北海道内に制限し、世界観の維持と無駄なレンダリングを防止。
- **ズーム連動型ドット間引き:** 広域表示（ズーム11以下）では未開拓ドットの描画をスキップし、驚異的な軽さ（サクサク感）を実現。拡大するとパパパッと未開拓店が出現します。
- **UGCを生む画像エクスポート:** 「📸 地図を画像として保存」ボタンにより、総開拓店舗数とアプリ名が透かし（ウォーターマーク）として刻印されたPNG画像を生成。SNS連携ボタンを置かずとも、自然なシェアを誘発します。

### 3. ダイナミック・マスタ合成 ＆ 記憶の地層UI
- **フィルターチップ:** 地図上の「☕️店内」「🥡テイクアウト」「🛍️物販」チップを切り替えるだけで、条件に合致する店舗を即座に絞り込み。
- **AIによる状態検知（動的マスタ更新）:** ユーザーの日記に「テイクアウト廃止」などの記載があれば、AIがそれを検知してシステムタグを付与。時系列順にデータを読み込むことで、**常に最新の店舗属性へ自動で上書き**されます。
- **エモーショナルな閉店処理:** カフェが閉店・移転した場合、データを削除せず温かみのあるセピア調の「🎞️ 思い出の地」としてマップに残り続けます（同じ場所に新店ができても、地層のようにポップアップ内に過去の店が併記されます）。

### 4. B2Bを見据えたデータパイプライン
- **匿名UUIDの自動付与:** ユーザーの端末（LocalStorage）に個人情報ゼロの匿名UUIDを自動生成してDB（Cloudflare D1）へ保存。将来の「同一ユーザーの回遊分析（B2B向け）」の価値を飛躍的に高めます。
- **2段階AIモデレーション:** 不適切画像はVision AIで即時破棄。テキストはLlama 3.1が多次元タグ（コーヒー、フード、環境）に構造化。未知の単語は裏でGASへ送信され、人間が監査する「ループエンジニアリング」を実装。

### 5. 自衛のDX（究極のプライバシー保護）
- 「🥡テイクアウト」や「🛍️豆・グッズ」を選択した場合、気象データのみを取得し**位置情報（緯度経度）は強制的に破棄**。自宅バレのリスクをシステムレベルで完全に排除。

## 🚀 未来の構想（ロードマップ）

### 🌉 フェーズ1.9：インフラ強靭化の最終仕上げ
- **画像ストレージの外部化:** DB（Cloudflare D1）の容量パンク（5GB制限）を防ぐため、画像の保存先をエグレス（転送料金）無料の「ConoHa オブジェクトストレージ（S3互換）」へ外部化。Cloudflare Workersから署名付きURL（Presigned URL）を用いたセキュアな直接アップロードを実装予定。

### 🏙️ フェーズ2：究極の完成形（静かなSNSとB2Bマネタイズ）
データ構造とインフラが安定したのち、「承認欲求を排除したSNS」と「店舗向けビジネス」を展開します。
1. **4つの「マルチマップ表示モード」**
   - ① **マイマップ**: 自分だけの訪問・開拓記録（現在の機能）。
   - ② **全員の開拓マップ**: 集合知を用いた「未開拓 / 誰かが行った / 自分も行った」の可視化。
   - ③ **属性フィルターマップ**: 「30代女性がよく行くカフェ」などの特化型表示。
   - ④ **「誰か」の軌跡マップ**: 匿名1名の過去のピンだけを表示する「すれちがい通信」的モード。
2. **B2B マネタイズ (EzoCafe for Business)**
   - 蓄積された「天候別」「時間帯別」「客層」「回遊ログ（UUID）」と、AIが抽出した「多次元タグ」を組み合わせ、**カフェ経営者向けの高度な商圏・顧客分析ダッシュボード**としてサブスクリプション提供します。
