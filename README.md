# ☕️ EzoCafe Log

## 📖 プロジェクト概要
既存SNSの「いいね」や「フォロワー」による疲弊感から脱却し、純粋なカフェ記録とデータ蓄積に特化したプロダクトです。
コンセプトは **「静寂」** と **「自衛のDX」**。
ユーザーには心理的安全性の高い秘密基地を提供しつつ、裏側ではAIがコンテンツを監視・構造化し、将来のB2Bデータビジネスに繋がる価値の高いカフェデータベースを自動精製します。

## 🛠 技術スタック (完全サーバーレスアーキテクチャ)
- **フロントエンド:** HTML / CSS / Vanilla JavaScript
- **バックエンド:** Cloudflare Workers
- **データベース:** Cloudflare D1 (SQLite)
- **AIモデル:** Cloudflare Workers AI
  - `llama-3.2-11b-vision-instruct` (画像解析)
  - `llama-3.1-8b-instruct` (テキスト解析)
- **外部連携API / ライブラリ:**
  - `exifr` (Exif情報抽出)
  - `Open-Meteo API` (過去・現在の気象データ取得)
  - `Leaflet.js` + `OpenStreetMap` (マップ描画)
  - `Chart.js` (データ分析・可視化)

## ✨ 現在の実装機能（フェーズ1: 基盤構築）

### 1. 2段階のAIモデレーション（画像＆テキスト）
- **Vision AI:** アップロード画像を監視し、カフェに無関係な自撮りや不適切画像を検知した場合は、システム側で画像データを自動破棄（容量節約と法務リスク回避）。
- **テキストAI:** 誹謗中傷やクレームを検知した場合、システムエラーで弾くのではなく、裏で「🚨共有不可」「🚨要確認」タグを付与して隔離。ユーザー画面にはAIタグを一切描画せず、検閲のストレスを与えません。

### 2. 超高効率なデータパイプライン
- 日記の自然な文章から、AIが「豆の産地」「精製プロセス」「農園」「席の位置」「設備環境」などを自動抽出し、分析用のタグとして裏側で蓄積します。

### 3. Exif解析 ＆ タイムトラベル天気取得
- 過去の写真をアップロードした際、Exifデータから「撮影日時」と「撮影位置」を抽出。さらに気象データAPI（Open-Meteo）を叩き、**当時の天気と気温を自動で割り出し**て記録します。

### 4. 究極のプライバシー保護（自衛のフェイルセーフ）
- 利用タイプで「🥡テイクアウト」や「🛍️豆・グッズ」を選択した場合、気象データのみを取得し、**位置情報（緯度経度）は強制的に破棄**してからデータベースへ送信。自宅バレのリスクをシステムレベルで排除しています。

### 5. マイマップ ＆ 分析ダッシュボード
- 記録した店舗はLeafletを用いたマップ上に自動マッピング。利用タイプ（店内/テイクアウト/物販）に応じてピンの絵文字が変化します。
- Chart.jsを用いたダッシュボードで、自分のカフェ開拓数や「よく行くカフェの傾向（タグ割合）」を可視化します。

## 🚀 未来の構想（ロードマップ）

### 🌉 フェーズ1.8：未来への架け橋
SNS化やビジネス展開に耐えうる、強靭なインフラとデータ構造を完成させます。
1. **マスタデータへの完全移行**
   - 抽出済みの北海道カフェマスタを使用し、GPSはあくまで「サジェスト（提案）」に留めることで、ピンズレと自宅バレを完全排除します。
2. **画像ストレージの外部化**
   - DBの容量パンク（クラウド破産）を防ぐため、エグレス（転送料金）無料の「ConoHa オブジェクトストレージ」へ移行。署名付きURLを用いたセキュアな直上げを実装します。
3. **開拓モード（ゲーミフィケーション）**
   - マスタ上の「未開拓のカフェ」を地図上にグレーで表示し、訪問すると色が変わるゲーム性を提供します。

### 🏙️ フェーズ2：究極の完成形（静かなSNSとB2Bマネタイズ）
システムが安定したのち、「承認欲求を排除したSNS」と「店舗向けビジネス」を展開します。
1. **4つの「マルチマップ表示モード」**
   - ① **マイマップ**: 自分だけの訪問記録。
   - ② **全員の開拓マップ**: 集合知を用いた「未開拓 / 誰かが行った / 自分も行った」の可視化。
   - ③ **属性フィルターマップ**: 「30代女性がよく行くカフェ」などの特化型表示。
   - ④ **「誰か」の軌跡マップ**: 匿名1名の過去のピンだけを表示する「すれちがい通信」的モード。
2. **自浄作用を持つマスタデータ**
   - ユーザーからの閉店報告（一定数で非表示）や、新店の遅延公開（API判定や複数人訪問で正式公開）により、管理コストをかけずにマスタを浄化します。
3. **B2B マネタイズ (EzoCafe for Business)**
   - これがプロジェクトの最大のゴールです。
   - 蓄積された「天候別」「時間帯別」「客層」「店内vsテイクアウト比率」と、AIが抽出した「席・目的タグ」を組み合わせ、**カフェ経営者向けの高度な分析ダッシュボード**として月額提供します。
