# ☕️ EzoCafe Log (エゾカフェ・ログ)

## 📖 プロジェクト概要 (Overview)

既存SNSの「いいね」や「フォロワー数」による精神的疲弊から脱却し、純粋なカフェ巡りの記録と、ユーザー自身の「足跡の蓄積」に特化した非同期型の探索SaaSプラットフォームです。

コンセプトは **「静寂」「自衛のDX」「記憶の地層」**、そして **「リアルタイムの気配」**。
承認欲求ノイズをシステムレベルで排除した心理的安全性の高い秘密基地を提供するとともに、裏側ではAIと匿名UUIDを駆使して、将来のB2Bマネタイズ（カフェ経営者向けデータ基盤）に繋がる高純度のデータベースを自動精製します。

---

## 🛠 技術スタック (完全サーバーレス・エッジアーキテクチャ)

* **Frontend:** HTML5 / CSS3 / Vanilla JavaScript (ES6+ モジュール設計) / Canvas API
* **Map Engine:** Leaflet.js, OpenStreetMap
* **Backend / Edge:** Cloudflare Workers
* **Database:** Cloudflare D1 (SQLite)
* **AI (Cloudflare Workers AI):** `@cf/meta/llama-3.1-8b-instruct`
* ※コストとパフォーマンス最適化のため重いVision AI（画像検閲）を廃止し、テキストAIによる「インサイト抽出」へリソースを全集中。


* **Utilities:** `exifr` (EXIF解析), `html2canvas` (マップ画像エクスポート), `Chart.js` (データ可視化)

## ✨ 現在の実装内容（Phase 3.5: フロントエンドの堅牢化とUXの極致）

### 1. フロントエンドのモジュール化とアーキテクチャ刷新

* 肥大化していた単一ファイルを役割ごとに分割（`map.js`, `form.js`, `list.js`, `api.js`, `state.js`）。状態管理とUI描画、通信層を分離する堅牢な設計へ移行し、今後の保守性と拡張性を極限まで高めました。

### 2. 直感と精度を両立するスマート・ピッカー

* **住所・市区町村検索 (Nominatim API):** 手動マップピッカーに検索バーを搭載。キーワード入力から目標地点へ `flyTo` メソッドで滑らかに遷移するアニメーションを実装。
* **フリクションレスなUX:** 常にマップ中央に追従する「ターゲットピン」を配置。どこを記録しようとしているかの認知負荷を下げ、座標のズレを視覚的に防ぐ徹底したポカヨケを実装しています。
* **インテント（意図）のトラッキング:** 「自宅でのテイクアウト写真」からは自動で座標を破棄し（自宅バレ防止）、手動で刺したピンは意図的な記録として座標を保持する高度なロケーション管理。

### 3. 記憶の地層のエクスポート（高画質マップ画像生成）

* **自己完結する達成感:** 他者へのアピールではなく、自分自身の軌跡を俯瞰・保存するためのエクスポート機能。
* **動的ウォーターマーク:** 記録時のズームUI等を自動で非表示にし、「現在の総開拓店舗数」が刻まれた透かしロゴを合成。`html2canvas` により、その瞬間のマップ状態を高解像度のPNG画像としてローカルに保存します。

### 4. リアルタイム「気配」の共有（揮発性ステータス）

* **「🔥 今、混んでる！」ボタン:** マップ上のボトムシートから、現在の混雑状況をワンタップで報告。
* **30分で消える熱量:** Cloudflare D1にTTLを持たせ、報告された店舗のピンは赤く脈打ち（Pulse Animation）、30分で自動的に元の静寂なマップへ戻ります（1時間以内の連続タップはバックエンドで遮断）。

### 5. アノニマストレース（ゴーストピン）

* **究極の非同期体験:** Premiumユーザー専用機能。他者が過去に訪れた記録が、紫色のオーラを放つ「ゴーストピン」としてマップに出現。
* **文学的マスキング:** 生の訪問日時は「ある夏の午後」などに抽象化され、AIが抽出した客観的なタグのみが共有される「引き算のUX」を体現。

### 6. 📊 B2B 店舗全体アナリティクスダッシュボード

* **即戦力のマーケティングデータ:** バックエンドの集計エンジンにより、店舗に紐づく全ユーザーの日記から「性別」「年代」「客観的イメージ（AIタグ）」を瞬時に集計。
* **権限による段階的開示 (RBAC):** `admin` または `business` 権限を持つユーザーがアクセスした時にのみ、グラフ化された顧客データが展開されます。

## 🚫 削除・排除した機能（やらないこと）

本アプリのコンセプトを守るため、以下の機能は意図的に排除・削除されています。

* **SNS連携・シェアボタン・いいねの完全排除:** 「他者からの評価を前提としたUI」は承認欲求を刺激し疲弊を生むため実装しません。
* **プロフィール・ユーザー名の排除:** 誰がピンを立てたかは一切表示しません。顔が見えないからこそ想像力が掻き立てられる、極上の非同期体験を徹底します。
* **過剰防衛の排除:** ユーザーのオプトイン（自己責任に基づく公開設定）を信頼し、システムの過剰な自動検閲（自撮り弾きや警告ポップアップ）を撤廃。サクサクと直感的に記録できるUXを優先しています。

## 🚀 未来の構想（ロードマップ）

### ☁️ Phase 4.0: 画像インフラのS3互換ストレージ外部化

* Cloudflare D1の容量制限（5GB）を突破するため、画像の保存先を「ConoHa オブジェクトストレージ（S3互換）」等へ外部化。Cloudflare Workersから署名付きURL（Presigned URL）を用いたセキュアなダイレクトアップロードアーキテクチャを導入します。

### 🏢 Phase 5.0: B2B/B2C マネタイズとPoCの開始

* **B2B向けリアルタイム集客:** 店舗オーナー自身が「🈳 空席出ました！」「🍪 焼き菓子焼き上がりました！」というリアルタイムピンを30分限定でマップに落とせる機能を展開。
* **B2Cアップセル:** Freeユーザーのマップ上にもゴーストピンを表示させつつ、タップ時の情報を「すりガラスUI」で隠し、「月額課金でこの街の隠された足跡を解放しませんか？」というシームレスなコンバージョン動線を設計します。
