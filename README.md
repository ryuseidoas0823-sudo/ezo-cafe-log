# ☕️ EzoCafe Log

## 📖 プロジェクト概要
既存SNSの「いいね」や「フォロワー」による疲弊感から脱却し、純粋なカフェ記録とデータ蓄積に特化したプロダクトです。
コンセプトは **「静寂」** と **「自衛のDX」**。
ユーザーには心理的安全性の高い秘密基地とゲーミフィケーションを提供しつつ、裏側ではAIと人間（Human-in-the-Loop）が協調してコンテンツを監視・構造化し、将来のB2Bデータビジネスに繋がる価値の高いカフェデータベースを自動精製します。

## 🛠 技術スタック (完全サーバーレス・エッジアーキテクチャ)
- **フロントエンド:** HTML / CSS / Vanilla JavaScript / Canvas API (画像圧縮)
- **バックエンド:** Cloudflare Workers
- **データベース:** Cloudflare D1 (SQLite)
- **AIモデル:** Cloudflare Workers AI
  - `llama-3.2-11b-vision-instruct` (画像解析・モデレーション)
  - `llama-3.1-8b-instruct` (テキスト解析・多次元タグJSON構造化)
- **外部連携API / ツール:**
  - `Google Apps Script (GAS)` (AI監査用Webhook・スプレッドシート連携)
  - `exifr` (Exif情報抽出)
  - `Open-Meteo API` (過去・現在の気象データ取得)
  - `Leaflet.js` + `OpenStreetMap` (マップ描画)
  - `Chart.js` (データ分析・可視化)

## ✨ 現在の実装機能（フェーズ1.8: 基盤・開拓モード完成）

### 1. アクション誘導型 UI/UX と超高効率パイプライン
- **画像ファースト入力:** 写真をアップロードした瞬間に、クライアント側（Canvas）で長辺800px・JPEG80%に自動リサイズ圧縮。通信量とクラウドストレージの圧迫を劇的に軽減します。
- **タイムトラベル天気取得:** 写真のExifデータから「撮影日時」と「位置情報」を抽出し、気象データAPI（Open-Meteo）から**当時の天気と気温を自動入力**。ユーザーの認知負荷を最小化します。

### 2. 2段階のAIモデレーション ＆ Human-in-the-Loop監査
- **Vision AI:** 自撮りや不適切画像を検知し、システム側で画像データを自動破棄（容量節約と法務リスク回避）。
- **テキストAI (JSON多次元構造化):** 自然な文章から「☕️コーヒー」「🍰フード」「🛋️雰囲気・設備」の次元ごとにタグを自動分類。
- **SPSエスカレーション:** AIが分類不能と判断した未知の単語やエラーは、裏側でコッソリGAS（Google Apps Script）へ送信。「AI監査ログ」スプレッドシートに蓄積され、人間が継続的にプロンプトをチューニングできる「ループエンジニアリング」を実現しています。

### 3. 究極のプライバシー保護（自衛のフェイルセーフ）
- 利用タイプで「🥡テイクアウト」や「🛍️豆・グッズ」を選択した場合、気象データのみを取得し、**位置情報（緯度経度）は強制的に破棄**してからDBへ送信。自宅バレのリスクをシステムレベルで排除しています。

### 4. 開拓モード（ゲーミフィケーション・マップ）
- 店舗マスタデータをベースとしてマップ上に「🏳️未開拓（グレー📍）」のピンを敷き詰めます。
- ユーザーが記録を行うと、その場所が「☕️訪問済み（特徴タグ色）」や「💭行きたい（オレンジ）」に上書き（マージ）される陣取りゲームのような体験を提供。再訪モチベーションを極限まで高めます。

### 5. ノイズレス・アナリティクス
- 「利用タイプ（行動）」と「好みのタグ（嗜好）」のグラフを完全に分離表示。
- 「💭行きたいお店」や「📦未整理の写真（一括アップロード用空データ）」はアナリティクスの母数から自動除外され、B2B提供に耐えうる純度100%の行動ログのみを可視化します。

## 🚀 未来の構想（ロードマップ）

### 🌉 フェーズ1.9：インフラ強靭化の最終仕上げ
DBの容量パンク（クラウド破産）を防ぐため、画像の保存先をエグレス（転送料金）無料の「ConoHa オブジェクトストレージ」へ外部化。署名付きURLを用いたセキュアな直上げを実装します。

### 🏙️ フェーズ2：究極の完成形（静かなSNSとB2Bマネタイズ）
システムとデータ構造が安定したのち、「承認欲求を排除したSNS」と「店舗向けビジネス」を展開します。
1. **4つの「マルチマップ表示モード」**
   - ① **マイマップ**: 自分だけの訪問・開拓記録。
   - ② **全員の開拓マップ**: 集合知を用いた「未開拓 / 誰かが行った / 自分も行った」の可視化。
   - ③ **属性フィルターマップ**: 「30代女性がよく行くカフェ」などの特化型表示。
   - ④ **「誰か」の軌跡マップ**: 匿名1名の過去のピンだけを表示する「すれちがい通信」的モード。
2. **自浄作用を持つマスタデータ**
   - ユーザーからの閉店報告（一定数で非表示）や、新店の遅延公開（API判定や複数人訪問で正式公開）により、管理コストをかけずにマスタを浄化します。
3. **B2B マネタイズ (EzoCafe for Business)**
   - 蓄積された「天候別」「時間帯別」「客層」「店内vsテイクアウト比率」と、AIが抽出した「多次元タグ」を組み合わせ、**カフェ経営者向けの高度な商圏・顧客分析ダッシュボード**として月額提供します。これが本プロジェクトの最大のゴールです。
