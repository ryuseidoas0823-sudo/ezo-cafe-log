# ☕️ EzoCafe Log (エゾカフェ・ログ)

## 📖 プロジェクト概要 (Overview)
既存SNSの「いいね」や「フォロワー数」による精神的疲弊から脱却し、純粋なカフェ巡りの記録と、ユーザー自身の「足跡の蓄積」に特化した非同期型の探索SaaSプラットフォームです。

コンセプトは **「静寂」「自衛のDX」「記憶の地層」**、そして **「リアルタイムの気配」**。
承認欲求ノイズをシステムレベルで排除した心理的安全性の高い秘密基地を提供するとともに、裏側ではAIと匿名UUIDを駆使して、将来のB2Bマネタイズ（カフェ経営者向けデータ基盤）に繋がる高純度のデータベースを自動精製します。

---

## 🛠 技術スタック (完全サーバーレス・エッジアーキテクチャ)
- **Frontend:** HTML5 / CSS3 / Vanilla JavaScript / Canvas API
- **Map Engine:** Leaflet.js, OpenStreetMap
- **Backend / Edge:** Cloudflare Workers
- **Database:** Cloudflare D1 (SQLite)
- **AI (Cloudflare Workers AI):** `@cf/meta/llama-3.1-8b-instruct`
  - ※フェーズ3.0にて、コストとパフォーマンスの最適化のため重いVision AI（画像検閲）を廃止し、テキストAIによる「インサイト抽出」へリソースを全集中させています。
- **Utilities:** `exifr` (EXIF解析), `html2canvas` (ウォーターマーク画像生成), `Chart.js` (データ可視化)

## ✨ 現在の実装内容（Phase 3.1: 気配の共有と自衛のDXの極致）

### 1. リアルタイム「気配」の共有（揮発性ステータス）
- **「🔥 今、混んでる！」ボタン:** マップ上のボトムシートから、現在の混雑状況をワンタップで報告。
- **30分で消える熱量:** Cloudflare D1にTTL（有効期限）を持たせた専用テーブルを構築。報告された店舗のピンはマップ上で赤く脈打ち（Pulse Animation）、30分経過すると自動的に静寂なマップへと戻ります。
- **スパム防止（ポカヨケ）:** 同一ユーザー（UUID）からの1時間以内の連続報告をバックエンドでブロック。

### 2. アノニマストレース（ゴーストピン）
- **究極の非同期体験:** Premiumユーザー専用機能。他者が過去に訪れた記録が、紫色のオーラを放つ「ゴーストピン」としてマップに出現します。
- **文学的マスキング:** 生の訪問日時は「ある夏の午後」などに抽象化され、AIが抽出した客観的なタグのみが共有される、彻底した「引き算のUX」を体現しています。

### 3. スマート・マスキングと手動マップピッカー
- **インテント（意図）のトラッキング:** 「自宅でのテイクアウト写真」からは自動で座標を破棄し（自宅バレ防止）、「ユーザーが手動でマップに刺したピン」は意図的な記録として座標を保持する、極めて高度なロケーション管理を実装。
- **堅牢なエラーハンドリング:** JavaScript特有の「`null`文字列化」によるマップ描画クラッシュを、フロント/バック両面の厳格なバリデーションで完全に根絶。

### 4. 📊 B2B 店舗全体アナリティクスダッシュボード
- **即戦力のマーケティングデータ:** バックエンドの集計エンジンにより、店舗に紐づく全ユーザーの日記から「性別」「年代」「客観的イメージ（AIタグ）」を瞬時に集計。
- **権限による段階的開示 (RBAC):** `admin` または `business` 権限を持つユーザーがボトムシートを開いた時にのみ、グラフ化された顧客データが展開されます。

## 🚫 削除・排除した機能（やらないこと）

本アプリのコンセプトを守るため、以下の機能は意図的に排除・削除されています。

- **SNS連携・シェアボタン・いいねの完全排除:** 「他者からの評価を前提としたUI」は承認欲求を刺激し疲弊を生むため実装しません。
- **プロフィール・ユーザー名の排除:** 誰がピンを立てたかは一切表示しません。顔が見えないからこそ想像力が掻き立てられる、極上の非同期体験を徹底します。
- **過剰防衛の排除:** ユーザーのオプトイン（自己責任に基づく公開設定）を信頼し、システムの過剰な自動検閲（自撮り弾きや警告ポップアップ）を撤廃。サクサクと直感的に記録できるUXを優先しています。

## 🚀 未来の構想（ロードマップ）

### 🧩 Phase 3.5: フロントエンドのモジュール化 (リファクタリング)
- 肥大化した `main.js` や `map.js` を、「画像処理」「UI描画」「フォーム制御」「通信」などの関心ごとに分割（ES Modules化）。今後の保守性と拡張性を極限まで高めます。

### ☁️ Phase 4.0: 画像インフラのS3互換ストレージ外部化
- Cloudflare D1の容量制限（5GB）を突破するため、画像の保存先を「ConoHa オブジェクトストレージ（S3互換）」等へ外部化。Cloudflare Workersから署名付きURL（Presigned URL）を用いたセキュアなダイレクトアップロードアーキテクチャを導入します。

### 🏢 Phase 5.0: B2B/B2C マネタイズとPoCの開始
- **B2B向けリアルタイム集客:** 店舗オーナー自身が「🈳 空席出ました！」「🍪 焼き菓子焼き上がりました！」というリアルタイムピンを30分限定でマップに落とせる機能を展開。
- **B2Cアップセル:** Freeユーザーのマップ上にもゴーストピンを表示させつつ、タップ時の情報を「すりガラスUI」で隠し、「月額課金でこの街の隠された足跡を解放しませんか？」というシームレスなコンバージョン動線を設計します。
