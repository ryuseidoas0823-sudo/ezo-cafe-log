# ☕️ EzoCafe Log (エゾカフェ・ログ)

## 📖 プロジェクト概要
既存SNSの「いいね」や「フォロワー数」による精神的疲弊から脱却し、純粋なカフェ巡りの記録と、ユーザー自身の「足跡の蓄積」に特化したプロダクトです。
コンセプトは **「静寂」「自衛のDX」「記憶の地層」**。
ユーザーには承認欲求を煽らない心理的安全性の高い秘密基地と、圧倒的なサクサク感を提供しつつ、裏側ではAIと匿名UUIDを駆使して、将来のB2Bビジネス（カフェ経営者向けデータ基盤）に繋がる高純度のデータベースを自動精製します。

---

## 🛠 技術スタック (完全サーバーレス・エッジアーキテクチャ)
- **フロントエンド:** HTML / CSS / Vanilla JavaScript / Canvas API (画像圧縮・タイポグラフィ生成・マップ画像化)
- **バックエンド:** Cloudflare Workers
- **データベース:** Cloudflare D1 (SQLite)
- **AIモデル (Cloudflare Workers AI):**
  - `llama-3.2-11b-vision-instruct` (画像解析・不適切画像の自動破棄)
  - `llama-3.1-8b-instruct` (テキスト解析・多次元タグのJSON構造化・状態変化の検知)
- **外部連携API / ツール:**
  - `html2canvas` (マップのウォーターマーク付き画像出力)
  - `exifr` (Exif情報抽出)
  - `Open-Meteo API` (過去・現在の気象データ自動取得)
  - `Leaflet.js` + `OpenStreetMap` (マップ描画・LOD UI制御)
  - `Chart.js` (データ分析・可視化)

## ✨ 現在の実装内容（フェーズ1.8: UX/UI・開拓モードの極致）

### 1. 段階的開示UIと「写真なし記録」の最適解
- **段階的開示 (Progressive Disclosure):** 画面を開いた瞬間は「写真を追加」のみ。写真を選ぶとフワッと入力フォームが展開し、ユーザーの認知負荷を極限まで下げます。
- **タイポグラフィカードの自動生成:** 写真がない場合（豆だけ買った時など）、履歴画面に文字だけの寂しいログが並ばないよう、Canvas APIを用いて「タグカラーをベースにした美しいタイポグラフィ画像」をフロントエンドで自動生成します（DB容量消費ゼロ）。

### 2. LOD（遅延レンダリング）搭載の超軽量マップ
- **北海道完全ロック:** マップの表示範囲を北海道内に制限し、世界観の維持と無駄なレンダリングを防止。
- **ズーム連動型ドット間引き (LOD):** 広域表示（ズーム11以下）では未開拓ドットの描画をスキップし、驚異的な軽さ（サクサク感）を実現。拡大すると未開拓店がパパパッと出現します。

### 3. パーソナル・カフェCRMと「記憶の地層」
- **AI気分タグフィルター:** ユーザーが入力した文字だけでなく、AIが抽出した「🤖 ゆったり」「🤖 作業」といった目的タグでマップを絞り込み、今の気分に最適なカフェを瞬時に探し出せます。
- **動的マスタ更新 (Dynamic Master):** ユーザーの日記に「テイクアウト廃止」などの記載があれば、AIが検知してシステムタグを付与。時系列順にデータを読み込むことで、常に最新の店舗属性へ自動で上書きされます。
- **エモーショナルな閉店処理:** カフェが閉店・移転した場合、データを削除せず温かみのあるセピア調の「🎞️ 思い出の地」としてマップに残り続けます（同じ場所に新店ができても、地層のようにポップアップ内に過去の店が併記されます）。

### 4. B2Bを見据えたデータパイプライン
- **匿名UUIDの自動付与:** ユーザーの端末（LocalStorage）に個人情報ゼロの匿名UUIDを自動生成してDBへ保存。将来の「同一ユーザーの回遊分析」の価値を飛躍的に高めます。
- **2段階AIモデレーション:** 不適切画像はVision AIで即時破棄。未知の単語は裏でGASへ送信され、人間が監査する「ループエンジニアリング」を実装。

## 🚫 削除・排除した機能（やらないこと）

本アプリのコンセプト「静寂」と「自衛のDX」を守るため、以下の機能は意図的に排除・削除されています。

- **SNS連携・シェアボタンの完全排除:** 「シェアすることを前提としたUI」は承認欲求を刺激するため実装しません。代わりに、ウォーターマーク（総開拓数とロゴ）が入った「マップの画像保存機能」のみを提供し、ユーザーの自発的なUGC（口コミ）に委ねます。
- **プロフィール・ユーザー名の排除:** 将来「全員の開拓マップ」を実装する際も、誰がそのピンを立てたかは一切表示しません。「すれちがい通信」のような匿名環境を徹底します。
- **GPSによる直接的な位置情報保存の廃止:** 自宅バレを防ぐため、記録は「店舗マスタからの検索・選択」を正とします。さらに「テイクアウト」「物販」を選んだ場合は、緯度経度のデータを強制的に破棄（null化）する強力なポカヨケを実装しています。

## 🚀 未来の構想（ロードマップ）

### 🌉 フェーズ1.9：インフラ強靭化の最終仕上げ
- **画像ストレージの外部化:** DB（Cloudflare D1）の容量パンク（5GB制限）を防ぐため、画像の保存先をエグレス（転送料金）無料の「ConoHa オブジェクトストレージ（S3互換）」へ外部化。Cloudflare Workersから署名付きURL（Presigned URL）を用いたセキュアな直接アップロードを実装予定。

### 🏙️ フェーズ2：究極の完成形（静かなSNSとB2Bマネタイズ）
データ構造とインフラが安定したのち、「承認欲求を排除したSNS」と「店舗向けビジネス」を展開します。
1. **4つの「マルチマップ表示モード」**
   - ① **マイマップ**: 自分だけの訪問・開拓記録（現在の機能）。
   - ② **全員の開拓マップ**: 集合知を用いた「未開拓 / 誰かが行った / 自分も行った」の可視化。
   - ③ **属性フィルターマップ**: 「30代女性がよく行くカフェ」などの特化型表示。
   - ④ **「誰か」の軌跡マップ**: 匿名1名の過去のピンだけを表示するエモさ特化モード。
2. **B2B マネタイズ (EzoCafe for Business)**
   - 蓄積された「天候別」「時間帯別」「客層」「回遊ログ（UUID）」と、AIが抽出した「多次元タグ」を組み合わせ、**カフェ経営者向けの高度な商圏・顧客分析ダッシュボード**としてサブスクリプション提供します。
